# Инструкция для нового заведения

Эту инструкцию отправляйте вместе с папкой проекта.  
Заведению нужны только: **Google-аккаунт**, **компьютер/планшет с камерой** и **15 минут**.

---

## Что в комплекте

| Файл / папка | Назначение |
|--------------|------------|
| `index.html` | Страница киоска (главный файл) |
| `models/` | Модели распознавания лиц (не удалять!) |
| `apps-script/Code.gs` | Код для Google Таблицы (база данных) |
| `НАСТРОЙКА.md` | Эта инструкция |

---

## Часть 1. Google Таблица (5 минут)

### Шаг 1. Создайте таблицу

1. Откройте [Google Таблицы](https://sheets.google.com)
2. Нажмите **+ Пустой файл**
3. Назовите таблицу, например: **«Столовая — учёт питания»**

### Шаг 2. Откройте Apps Script

1. В таблице: меню **Расширения → Apps Script**
2. Откроется редактор кода
3. Удалите всё содержимое файла `Code.gs`
4. Откройте файл **`apps-script/Code.gs`** из комплекта на компьютере
5. Скопируйте весь код и вставьте в редактор Google
6. Нажмите **Сохранить** (иконка дискеты)

### Шаг 3. Создайте листы в таблице

1. В редакторе Apps Script сверху выберите функцию **`setupSheets`**
2. Нажмите **Выполнить** ▶
3. Google попросит разрешения:
   - **Разрешить** → выберите свой аккаунт
   - «Приложение не проверено» → **Дополнительные настройки** → **Перейти на страницу...** → **Разрешить**
4. В таблице появятся листы: **Employees**, **Meals**, **Logs**

### Шаг 4. Опубликуйте веб-приложение

1. В Apps Script: **Развернуть → Новое развертывание**
2. Нажмите шестерёнку ⚙ → выберите **Веб-приложение**
3. Заполните:
   - **Описание:** Киоск питания
   - **Запуск от имени:** Я
   - **У кого есть доступ:** Все
4. Нажмите **Развернуть**
5. **Скопируйте URL** — он заканчивается на `/exec`  
   Пример: `https://script.google.com/macros/s/ABC123.../exec`

> Сохраните этот URL — он понадобится в части 2.

## Часть 2. Размещение киоска (5 минут)

Выберите один из вариантов.

### Вариант А — на компьютере в локальной сети (проще всего)

1. Скопируйте всю папку проекта на компьютер у столовой
2. Откройте терминал / командную строку в этой папке
3. Выполните:

```bash
python3 -m http.server 8080
```

4. На планшете/ПК откройте браузер: **http://IP_КОМПЬЮТЕРА:8080**  
   (IP можно узнать: Windows — `ipconfig`, Mac — `ifconfig`)

### Вариант Б — бесплатный хостинг (GitHub Pages)

1. Загрузите папку на GitHub
2. Settings → Pages → Source: main branch
3. Откройте ссылку вида `https://ваш-логин.github.io/mc/`

> Для камеры нужен HTTPS — GitHub Pages даёт его автоматически.

## Часть 3. Первый запуск киоска (3 минуты)

1. Откройте страницу киоска в **Chrome** или **Edge**
2. Разрешите доступ к **камере**
3. Откроется мастер настройки — пройдите 3 шага:
   - Вставьте **URL Apps Script** (из части 1, шаг 4)
   - Укажите **название точки** (например: «Столовая цех №2»)
   - Нажмите **Проверить** → **Готово**
4. Нажмите **F11** для полноэкранного режима

## Ежедневная работа

### Для сотрудника (есть Face ID)

1. Подойти к камере
2. Система распознает автоматически (2–3 секунды)
3. Нажать **Завтрак**, **Обед** или **Ужин**
4. Готово — запись в Google Таблице на листе **Meals**

### Для нового сотрудника (нет Face ID)

1. Подойти к камере
2. Если лицо не распознано — появится форма **«Быстрая регистрация»**
3. Ввести **ФИО** → нажать **«Зарегистрировать и продолжить»**
4. Face ID создаётся автоматически с камеры
5. Сразу нажать **Обед** (или другой приём пищи)

> Повторная регистрация того же типа питания в один день **заблокирована**.

## Просмотр журнала

Откройте Google Таблицу → лист **Meals**:

| Столбец | Значение |
|---------|----------|
| timestamp | Дата и время |
| employeeName | ФИО |
| mealType | Завтрак / Обед / Ужин |
| siteName | Название точки |
| matchScore | Точность Face ID (%) |

Или в киоске: **Администрирование → Журнал**.

## Частые проблемы

| Проблема | Решение |
|----------|---------|
| «API не отвечает» | Проверьте URL, доступ «Все пользователи», нажмите «Обновить данные» |
| «Модели не загружаются» | Открывайте через сервер (`python3 -m http.server`), не двойным кликом |
| Камера не работает | Нужен HTTPS или localhost; разрешите камеру в браузере |
| Лицо не распознаётся | Хорошее освещение, лицо прямо в камеру; или быстрая регистрация |
| Обед уже выдан | Один тип питания — один раз в день (это нормально) |
