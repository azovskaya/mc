# Киоск питания — учёт по Face ID

**Главное:** распознавание лица → фиксация **кто**, **когда** и **что** поел (завтрак / обед / ужин) в Google Таблице.

---

## Что отправить новому заведению

Упакуйте в ZIP всю папку проекта:

```
mc.zip
├── index.html          ← страница киоска
├── models/             ← не удалять!
├── apps-script/Code.gs ← код для Google
├── НАСТРОЙКА.md        ← инструкция для них
└── README.md
```

Заведение: **Google-аккаунт + планшет с камерой + 15 минут**.

Подробная пошаговая инструкция для них — в файле **[НАСТРОЙКА.md](НАСТРОЙКА.md)**.

---

## Настройка Google (кратко)

| Шаг | Действие |
|-----|----------|
| 1 | [Создать Google Таблицу](https://sheets.google.com) |
| 2 | Расширения → Apps Script → вставить `apps-script/Code.gs` |
| 3 | Выполнить функцию **`setupSheets`** |
| 4 | Развернуть → Веб-приложение → доступ **Все** |
| 5 | Скопировать URL `/exec` → вставить в киоск |

При первом открытии киоска появится **мастер настройки** (3 шага) с кнопкой «Скопировать код Apps Script».

## Как работает киоск

### Есть Face ID (уже в базе)
1. Сотрудник подходит к камере
2. Система распознаёт автоматически
3. Нажимает **Обед** (или завтрак/ужин)
4. Запись в Google Таблице → лист **Meals**

### Нет Face ID (новый сотрудник)
1. Лицо не распознано → появляется **«Быстрая регистрация»**
2. Вводит **ФИО** → «Зарегистрировать и продолжить»
3. Face ID создаётся с камеры **за 5 секунд**
4. Сразу нажимает **Обед** — без перехода в админку

Кнопка **«Новый сотрудник»** — регистрация вручную в любой момент.

## Запуск страницы

```bash
cd mc
python3 -m http.server 8080
```

Открыть: **http://localhost:8080** (или разместить на GitHub Pages / любом хостинге с HTTPS).

> Не открывайте `index.html` двойным кликом — модели Face ID не загрузятся.

---

## Журнал «кто когда пообедал»

- **В Google:** лист **Meals** — timestamp, employeeName, mealType, siteName
- **В киоске:** Администрирование → Журнал / Отчёты / Экспорт CSV

Повторная выдача одного типа питания в день **блокируется**.
